# Sprint G — Guided Parent Onboarding and Document-First Uploads

**Status:** Planned
**Window:** Next sprint

## Goal
Make the parent journey calm, obvious, and reassuring at the start by explaining the full process up front, showing the required-document checklist clearly, and guiding uploads in a simple, step-by-step flow that confirms the right documents have been captured without surprise follow-on steps.

## Why this sprint
The current parent experience already has strong guidance and structure, but the next improvement is about pacing. A first-time parent should know what the whole process looks like before they begin, what they must have ready, and how the upload steps will work, so they never feel like the application keeps expanding forever.

## Scope
### P0
1. Clarify the opening parent screen so the whole journey is explained before the form begins
2. Add a clear required-document checklist at the outset
3. Distinguish required, conditional, and optional documents with plain language
4. Guide the upload flow one step at a time with confirmation after each upload
5. Make it obvious when the application is complete enough to continue

### P1
1. Reduce visual noise in the parent shell and supporting cards
2. Tighten helper copy so it is short, calm, and directive
3. Improve the upload confirmation and error recovery states
4. Make the journey easier to follow on mobile without adding extra steps

### P2
1. Small copy refinement from browser verification
2. Minor spacing and card-density polish where the flow still feels crowded

## Deliverables
1. Parent landing and checklist section that clearly explains the full process
2. A document readiness panel that acts like a tickbox checklist before upload begins
3. Step-by-step upload progression with clear “done” signals
4. Revised guidance cards that help without overwhelming
5. Local docs and handoff updated to reflect the new parent-first sprint

## Acceptance Criteria
1. A first-time parent can understand the whole journey before starting
2. Required documents are visible up front and easy to prepare
3. Upload steps feel guided, not endless
4. The page never creates the sense that new requirements are being invented mid-flow
5. Parents always know what happens next and when they are ready to continue

## Exit Criteria
1. P0 parent-flow changes are implemented and verified in browser
2. The checklist and upload journey feel calm on desktop and mobile
3. Docs and handoff are updated to match the new sprint focus

## Current Truth
- The parent journey must explain the process before asking for action
- Required documents must be visible at the start
- Uploading must feel guided and finite, not open-ended
- Guidance should be concise and reassuring, not crowded
