# Sprint E — Admin Decision Intelligence and Guided Parent Flow

**Status:** Planned
**Window:** 2026-06-15 to 2026-06-26

## Goal
Turn the admin dashboard into a faster decision surface while keeping the parent upload workspace concise, guided, and easy to complete on mobile.

## Why this sprint
We have already improved theme consistency, parent upload guidance, queue filters, and basic analytics. Sprint E combines the strongest follow-up ideas into one cohesive pass so the product feels more operational and less like a set of separate screens.

## Scope
### P0
1. Admin decision intelligence
2. Admin trend comparisons and risk flags
3. Guided parent upload flow refinement

### P1
1. Stronger cross-screen visual consistency
2. Compact next-step guidance in the parent review flow
3. Better dashboard scanability on smaller screens

### P2
1. Small polish fixes from browser verification
2. Copy tightening where text remains verbose

## Deliverables
1. Admin dashboard decision panel with:
   - live queue snapshot
   - top risk flags
   - recent intake trend
   - clearer focus recommendations
2. Parent upload workspace with:
   - shorter instruction language
   - stronger step-by-step guidance
   - succinct action labels
3. Local docs and handoff updated to reflect the sprint

## Acceptance Criteria
1. Admin can identify blockers and fast-lane items in one glance.
2. Parent upload cards are short, direct, and guided.
3. The visual language remains consistent across auth, landing, parent, and admin screens.
4. The sprint tracker and handoff match what is actually in the repo.

## Exit Criteria
1. P0 items implemented and pushed.
2. Browser check completed on admin and parent flows.
3. Docs updated locally and in the repository.
