# Eunice School Intake Platform — Project Brief

## Executive Summary

This project proposes the design and development of a modern digital admissions and learner intake platform for South African schools, beginning with a pilot implementation using Eunice High School as the initial case study.

The current admissions process is highly manual and operationally inefficient, relying heavily on:
- Google Forms
- Email communication
- Spreadsheet tracking
- Manual document verification
- Repeated follow-ups
- Fragmented storage processes

The project seeks to:
- Simplify the parent admissions experience
- Reduce administrative burden
- Automate repetitive communication
- Centralise admissions management
- Improve workflow visibility
- Establish foundations for a future scalable SaaS civic-tech platform for broader South African school adoption

The initial implementation will focus on:
- Low-cost, lightweight, scalable, mobile-first workflow modernisation

---

## The Core Problem

**Current Operational Reality**

The current intake process is largely dependent on manual administration.

Admissions staff currently:
- Manually review every application
- Manually verify supporting documents
- Manually email parents regarding missing information
- Manually track application progress
- Manually organise uploaded files
- Manually communicate outcomes

This creates:
- Operational bottlenecks
- Duplicated effort
- Inconsistent compliance
- Administrative overload
- Poor visibility
- Frustrating parent experiences

## Initial Scope (Prototype / MVP)

The MVP phase will focus **ONLY** on:
- Learner applications
- Document uploads
- Workflow statuses
- Email communication
- Admissions administration

The platform is **NOT** initially intended to become:
- A full school ERP
- LMS
- Finance platform
- Student lifecycle system

**Scope discipline is critical.**

## Proposed User Experience

### Parent Experience
The system should provide:
- A clean mobile-first admissions portal
- Step-by-step guided applications
- Structured document uploads
- Application progress visibility
- Automated acknowledgements
- Clear status tracking

The experience should feel:
- Modern
- Simple
- Low-friction
- Mobile-friendly

Special consideration must be given to:
- Older Android devices
- Inconsistent internet connectivity
- Low technical literacy

### Admin Experience
Admissions administrators should be able to:
- View all applications centrally
- Filter and search applicants
- Review uploaded documents
- Flag missing information
- Update application statuses
- Trigger reminder emails
- Add internal notes
- Export reports

The platform should significantly reduce repetitive manual administration.

## Functional Requirements

### Parent Portal

**Core Features**
- Mobile-first application portal
- Multi-step application process
- Parent and learner data capture
- Required document uploads
- Save-and-return functionality
- Final review and submission
- Unique application reference number
- Application status tracking

### Automated Communication

**Initial Communication Scope**
- Submission acknowledgement
- Missing document reminders
- Application status updates
- Final acceptance/rejection notifications

Initial communication will use **email only**.  
WhatsApp integration may be introduced in later phases.

### Admin Dashboard

**Core Functions**
- View all applications
- Filter/search applicants
- Document review
- Missing document tracking
- Status management
- Internal notes
- Reminder triggers
- Data export functionality

### Document Management

**Requirements**
- Secure uploads
- Structured storage hierarchy
- Categorised document types
- Easy retrieval
- Audit timestamps
